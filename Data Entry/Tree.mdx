# Tree

Multi-level structured list.

## Scenes

Folders, organizational structures, biological classifications, countries and regions, etc., most of the structures in the world are tree structures. Using the tree control can fully display the hierarchical relationship, and has interactive functions such as expanding and collapsing selection.

- Display and edit several levels of product categories.

- Multiple choices of countries, provinces and cities.

## Example

```json
{
	"Tree": {
		"label": "Tree",
		"edit": {
			"type": "tree",
			"props": {
				"value": ":tree",
				"checkable": true,
				"remote": {
					"api": "/api/tree/search",
					"query": {
						"select": ["id", "name"]
					}
				}
			}
		}
	}
}
```

## Props

```ts
interface Option {
	key: string
	title: string
	children: Array<Option>
	disabled?: boolean
	selectable?: boolean
}

interface IProps extends TreeProps {
	options: array<Option>
	remote: {
		api: string
		query: {
			select: Array<string>
		}
	}
}
```

## API

| Properties | Description            | Type              | Default | Required | Version |
| ---------- | ---------------------- | ----------------- | ------- | -------- | ------- |
| value      | bind value             | string            | -       | yes      |         |
| options    | list of options        | IProps['options'] | -       | no       |         |
| remote     | list of remote options | [Remote](#Remote) | -       | no       |         |

### Remote

| Properties | Description | Type                      | Required |
| ---------- | ----------- | ------------------------- | -------- |
| api        | interface   | string                    | yes      |
| query      | interface   | IProps['remote']['query'] | no       |

For other properties, see [Antd Tree](https://ant.design/components/tree/)
